# VIYA Deployment Guide — Free Cloud Hosting

## 🏗️ Architecture
```
Frontend (Next.js)  →  Render.com (Free)
Backend  (FastAPI)  →  Render.com (Free)
Database (PostgreSQL) → Supabase (Free 500MB)
AI Engine → Google Gemini API (Free tier: 15 RPM)
```

| **Start Command** | `uvicorn main:app --host 0.0.0.0 --port $PORT` |

| Key | Value |

| `DATABASE_URL` | `postgresql://postgres:%5BLokiKavi%401211%5D@db.vsrwqziponygvashcuox.supabase.co:5432/postgres` |

## 🔄 Alternative Free Platforms

### Option A: Render.com ✅ (RECOMMENDED)
- **Pros**: Free, Python native, auto-deploy from GitHub, Singapore region
- **Cons**: Sleeps after 15 min idle (30s cold start)
- **Limits**: 750 hrs/month, 512MB RAM

### Option B: Railway.app
- **Pros**: $5 free credit/month, fastest deploy, great DX
- **Cons**: $5 credit runs out with heavy usage
- **Deploy**: `railway login` → `railway up`

### Option C: Koyeb.com
- **Pros**: Free tier, no cold starts, global edge
- **Cons**: 512MB RAM, limited build minutes
- **Deploy**: Connect GitHub → auto-deploy

### Option D: Fly.io
- **Pros**: Global edge, 3 shared VMs free
- **Cons**: Requires Docker, more complex setup
- **Deploy**: `fly launch` → `fly deploy`

### Option E: Vercel (Frontend Only)
- **Pros**: Best for Next.js, edge CDN, instant deploys
- **Cons**: Backend limited to 10s execution (too short for AI calls)
- **Best for**: Frontend only, use Render for backend

## 🔐 Security Checklist Before Going Live

- [ ] Change `SECRET_KEY` to a strong random value (Render auto-generates one)
- [ ] Set `GOOGLE_API_KEY` to your actual Gemini API key
- [ ] Verify Supabase RLS policies are active
- [ ] Test all critical endpoints: `/auth/register`, `/auth/login`, `/assessment/start`
- [ ] Verify `.env` and `.env.local` are in `.gitignore` (they are ✅)
- [ ] Never commit real API keys to Git
